* Gmail Integration
Gmail should be treated as a label-first provider. Its provider-native labels should map into Meeru's label model without forcing them through folder mappings, while Gmail system semantics such as Inbox, Trash, and Spam can still inform unified folder placement where appropriate.

** Gmail-Specific Features
- [ ] Integrate Gmail with the shared OAuth2 and credential infrastructure from plan 012
- [ ] Add Gmail label support:
  - [ ] Fetch all labels
  - [ ] Create/update/delete labels
  - [ ] Label hierarchy support
- [ ] Implement Gmail categories
- [ ] Add Gmail-specific search operators
- [ ] Support Gmail threading model

** Gmail Optimization
- [ ] Implement Gmail push notifications (IDLE)
- [ ] Add Gmail-specific batch operations
- [ ] Optimize unified folder derivation from Gmail system labels without collapsing user labels into folders
- [ ] Optimize provider-label mapping and sync performance
- [ ] Handle Gmail quota and limits
- [ ] Add Gmail settings sync

** Explicitly Out Of Scope
- Generic OAuth2 infrastructure shared across providers
- Generic IMAP and SMTP baseline behavior already owned by plan 003
- Cross-provider unified folder and label model owned by plan 006
