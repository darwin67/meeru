* Unified Folder and Label System
This plan owns Meeru's cross-provider organization model. Folders remain navigation containers, labels remain separate classification metadata, and the plan should define how both coexist without forcing label-first providers to masquerade as folder-only systems.

** Unified Container Model
- [ ] Design unified folder data model
- [ ] Design unified label data model
- [ ] Create folder mapping engine
- [ ] Create provider-label mapping model that remains separate from folder mappings
- [ ] Implement standard folder auto-detection:

  - [ ] Drafts mapping
  - [ ] Archive mapping
  - [ ] Trash mapping
- [ ] Build custom folder creation system
- [ ] Define how unified folders and unified labels interact in cross-account views without collapsing one into the other
- [ ] Add folder and label synchronization logic
