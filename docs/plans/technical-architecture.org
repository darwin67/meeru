* Data Flow Architecture

    UI->>Core: Request folder contents
    Core->>Storage: Check local cache

    alt Cache miss or stale
        Core->>SyncEngine: Request sync
        SyncEngine->>Provider: Fetch updates
        Provider-->>SyncEngine: Return emails

    Core->>AI: Process emails (async)
    AI-->>Core: Categories, priorities

    Core-->>UI: Display emails
    UI-->>User: Show folder view

* Sync Engine Architecture

#+begin_src mermaid
stateDiagram-v2
    [*] --> Idle

    Idle --> Checking: Sync triggered

    Checking --> LocalChanges: Has local changes
    Checking --> RemoteChanges: No local changes

    LocalChanges --> ApplyingLocal: Apply to remote
    ApplyingLocal --> RemoteChanges: Success
    ApplyingLocal --> ConflictResolution: Conflict detected

    RemoteChanges --> FetchingRemote: Fetch from provider
    FetchingRemote --> ApplyingRemote: Changes received
    FetchingRemote --> Idle: No changes

    ApplyingRemote --> Idle: Success
    ApplyingRemote --> ConflictResolution: Conflict detected

    ConflictResolution --> ResolvingConflicts: Process conflicts
    ResolvingConflicts --> ApplyingResolution: Resolution found
    ApplyingResolution --> Idle: Complete

    state ConflictResolution {
        [*] --> AnalyzeConflict
        AnalyzeConflict --> AutoResolve: Simple conflict
        AnalyzeConflict --> UserResolve: Complex conflict
        AutoResolve --> [*]
        UserResolve --> [*]
    }
#+end_src

** Performance Targets
*** Response Times
- [ ] Email list load: <100ms
- [ ] Email open: <50ms
- [ ] Search results: <500ms
- [ ] Folder switch: <200ms

*** Resource Usage
- [ ] Memory usage idle: <100MB
- [ ] Memory with 100k emails: <500MB
- [ ] CPU usage idle: <1%
- [ ] Disk cache size: Configurable limit

*** Scalability
- [ ] Support 1M+ emails per account
- [ ] Handle 20+ accounts simultaneously
- [ ] Background sync without UI impact
- [ ] Incremental search index updates
