* API Overview

The Meeru API provides programmatic access to email operations, enabling AI agents and automation scripts to interact with the email client. The API runs as a local HTTP server on the user's machine.

** Base URL
#+begin_example
http://localhost:8080/api/v1
#+end_example

** Authentication
All API requests require authentication using a bearer token:
#+begin_src http
Authorization: Bearer <api-token>
#+end_src

** Response Format
All responses are JSON with the following structure:
#+begin_src json
{
  "success": true,
  "data": { ... },
  "error": null,
  "meta": {
    "request_id": "uuid",
    "timestamp": "2026-03-30T12:00:00Z"
  }
}
#+end_src

* Email Operations Endpoints

** List Emails
Retrieve a list of emails with pagination and filtering.

*** Request
#+begin_src http
GET /emails
#+end_src

*** Query Parameters
| Parameter    | Type    | Required | Description                           |
|--------------+---------+----------+---------------------------------------|
| account_id   | string  | No       | Filter by account (all if omitted)    |
| folder_id    | string  | No       | Filter by folder                      |
| unread       | boolean | No       | Filter unread emails                  |
| starred      | boolean | No       | Filter starred emails                 |
| has_attachment | boolean | No     | Filter emails with attachments        |
| from         | string  | No       | Filter by sender                      |
| to           | string  | No       | Filter by recipient                   |
| subject      | string  | No       | Search in subject                     |
| date_from    | date    | No       | Start date (ISO 8601)                 |
| date_to      | date    | No       | End date (ISO 8601)                   |
| limit        | integer | No       | Results per page (default: 50)        |
| offset       | integer | No       | Pagination offset (default: 0)        |
| sort         | string  | No       | Sort field (date, from, subject)      |
| order        | string  | No       | Sort order (asc, desc)                |

** Common Error Codes
| Code                  | HTTP Status | Description                    |
|-----------------------+-------------+--------------------------------|
| UNAUTHORIZED          | 401         | Invalid or missing auth token  |
| FORBIDDEN             | 403         | Insufficient permissions       |
| NOT_FOUND             | 404         | Resource not found             |
| VALIDATION_ERROR      | 400         | Invalid request data           |
| RATE_LIMIT_EXCEEDED   | 429         | Too many requests              |
| INTERNAL_ERROR        | 500         | Server error                   |
| ACCOUNT_SYNC_REQUIRED | 409         | Account needs sync first       |
| AI_SERVICE_UNAVAILABLE| 503         | AI service temporarily down    |

* SDK Examples

** JavaScript/TypeScript
#+begin_src typescript
import { MeeruClient } from '@meeru/sdk';

const client = new MeeruClient({
  apiKey: 'your_api_key',
  baseUrl: 'http://localhost:8080/api/v1'
});

// List emails
const emails = await client.emails.list({
  unread: true,
  limit: 50
});

// Send email
const sent = await client.emails.send({
  accountId: 'acc_123',
  to: ['recipient@example.com'],
  subject: 'Hello from Meeru SDK',
  content: {
    text: 'This email was sent using the Meeru SDK!'
  }
});
#+end_src

** Python
#+begin_src python
from meeru import MeeruClient

client = MeeruClient(
    api_key='your_api_key',
    base_url='http://localhost:8080/api/v1'
)

# Search emails
results = client.search(
    query='from:boss@company.com',
    date_range={'from': '2026-01-01', 'to': '2026-03-30'}
)

# AI summarization
summary = client.ai.summarize(
    email_id='email_789',
    include_thread=True
)
#+end_src

** Rust
#+begin_src rust
use meeru_sdk::{MeeruClient, EmailFilter};

let client = MeeruClient::new(
    "your_api_key",
    "http://localhost:8080/api/v1"
)?;

// List unread emails
let emails = client.emails()
    .filter(EmailFilter::Unread)
    .limit(50)
    .list()
    .await?;

// Move to folder
client.emails()
    .id("email_789")
    .move_to("folder_archive")
    .await?;
#+end_src
